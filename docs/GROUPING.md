# Группировка содержимого группы (план)

> **Статус:** запланировано (ещё не реализовано).  
> Документ фиксирует требования, риски и технический подход **до** начала разработки — чтобы при неудаче можно было откатиться и не потерять контекст.

---

## Запрос пользователя

При выделении **группы** рядом с панелью управления должны появляться кнопки:

| Кнопка | Когда видна | Действие |
|--------|-------------|----------|
| **Группировать** | Группа выбрана, содержимое ещё не «прикреплено» | Всё, что внутри группы, становится единым блоком: перетаскивание группы двигает содержимое вместе с ней |
| **Разгруппировать** | После группировки | Содержимое возвращается на «первоначальный уровень» — карточки снова независимы на холсте |

Строки UI — **RU / EN** в `src/i18n/messages.ts`.

---

## Текущее поведение (baseline)

Сейчас группа в MindStorm — **только визуальная рамка**:

- Узлы **не** имеют `parentId` в React Flow.
- «Находится внутри группы» = карточка **геометрически** попадает в прямоугольник группы (`x`, `y`, `width`, `height`). Для resize группы используется критерий: **центр** карточки внутри bbox.
- **Уже реализовано (июнь 2026):** при изменении размера группы текстовые карточки внутри **масштабируются пропорционально** (`src/lib/groupResize.ts`). Это **не** то же самое, что кнопки «Группировать» / «Разгруппировать» ниже.
- JSON Canvas **не задаёт** явную иерархию parent-child ([спецификация](https://jsoncanvas.org/)): Obsidian тоже использует только координаты.
- Группа: `zIndex: -1`, карточки: `zIndex: 1`, рёбра: `z-index: 0` — **не ломать** без проверки.

Файлы: `MindCanvas.tsx`, `CardNodes.tsx` (`GroupCardNode`), `jsonCanvas.ts`, `index.css`.

## Целевое поведение (после реализации)

### Группировать

1. Пользователь выбирает группу → в `SelectionPanel` (или рядом) кнопка **Группировать**.
2. Находим карточки «внутри» группы (критерий — **см. решения ниже**, пока TBD).
3. Для каждой карточки:
   - `parentId` = `id` группы (React Flow sub-flow).
   - `position` переводится в **относительные** координаты от левого верхнего угла группы.
4. Перетаскивание группы двигает все прикреплённые карточки.
5. Связи между карточками внутри группы и с внешними узлами **должны остаться** (React Flow поддерживает edges при parent-child).
6. Операция попадает в **Undo/Redo** (`commitNow()` после изменения).

### Разгруппировать

1. Кнопка **Разгруппировать** вместо **Группировать** (или рядом), если у группы есть прикреплённые дети.
2. У детей снимается `parentId`, `position` переводится обратно в **абсолютные** координаты холста.
3. Undo/Redo — аналогично.

### Сохранение

Рекомендуемое расширение MindStorm (не ломает Obsidian):

```json
{
  "id": "card1",
  "type": "text",
  "x": 120,
  "y": 80,
  "width": 260,
  "height": 120,
  "text": "...",
  "parentGroupId": "g_product"
}
```

- В файле **всегда** хранятся абсолютные `x`/`y` (как сейчас) — для совместимости с `.canvas`.
- `parentGroupId` — опциональное поле MindStorm; при загрузке в Obsidian игнорируется.
- При `flowToCanvas` / `canvasToFlow` восстанавливать `parentId` в runtime.

Альтернатива (хуже): только runtime без сохранения — **не рекомендуется** (теряется при перезагрузке).

## Технический подход

### React Flow

- Встроенная поддержка **parent / child** через `node.parentId`.
- Позиция ребёнка — относительно родителя.
- Drag родителя двигает детей автоматически.
- Опции: `extent: 'parent'` — ограничить drag ребёнка рамкой группы (TBD, нужен выбор пользователя).

### Новые модули (план)

| Модуль | Назначение |
|--------|------------|
| `src/lib/groupMembership.ts` | «Кто внутри группы?», group / ungroup, конвертация координат |
| `SelectionPanel` / `Toolbar.tsx` | Кнопки Группировать / Разгруппировать |
| `jsonCanvas.ts` | `parentGroupId` ↔ `parentId` |
| `MindCanvas.tsx` | Обработчики, `commitNow`, состояние «группа уже сгруппирована» |

### Определение «внутри группы»

**Принято:** центр карточки внутри прямоугольника группы.

### Вложенные группы

**Принято:** группировать только **text**-карточки; другие группы внутри рамки не прикреплять.

### Drag после группировки

**Принято:** двигается **только группа целиком**; дети не перетаскиваются по отдельности (`draggable: false` на детях).

## Оценка безопасности

### Вердикт: **да, можно сделать безопасно**, если:

1. Реализовать **поэтапно** (MVP: одна группа, text-карточки, сохранение `parentGroupId`).
2. Не менять z-index правило «рёбра над группами» без визуальной проверки.
3. Каждый шаг проверять: drag, resize группы, edges, Undo, Save/Load, демо-схема.
4. При загрузке чистого Obsidian `.canvas` — работать как сейчас (без parentId, пока пользователь не нажмёт Группировать).

### Основные риски

| Риск | Митигация |
|------|------------|
| Ошибки в пересчёте координат absolute ↔ relative | Отдельные функции + ручная проверка на демо; unit-тесты по желанию |
| Рёбра «ломаются» при drag сгруппированного блока | Тест на демо-схeme; `syncEdgesWithSourceColors` после group/ungroup |
| Resize группы после группировки | Решить: дети масштабируются или только рамка (TBD) |
| Конфликт z-index parent/child | Не повышать z-index группы над рёбрами; проверить `index.css` |
| Undo после group/ungroup | `commitNow()` сразу после операции |
| Obsidian-совместимость | Абсолютные координаты в файле; `parentGroupId` — опциональное расширение |

### Что **не** входит в MVP (если не попросят отдельно)

- Авто-группировка при перетаскивании карточки в рамку.
- Группировка нескольких выделенных групп разом.
- Клиппинг содержимого по границе группы (Obsidian groups не clip).
